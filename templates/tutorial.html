Layout

The below image shows the layout of the MedEx on initial load. There is a sidebar which provides the filter options and the main layout provides several option to display the data in various graphical representations.



Patient Filter


Filters are available for every tab. The filtering can be done either on basis of the patient visit or only through the entity selection.

The Visit field have multiple options like baseline along with several patient follow-ups. The user is allowed to not filter the patients based on a visit by selecting the None option.

The Entity field allows the user to filter the patient data based on the type of data items such as numerical, categorical and date items.

Depending upon the type of data item selected in the Entity field, respective filter panel is displayed for further filtering options. Multiple filters can be applied as shown in Example 2. The result also shows the number of patients remaining after applying the filters.



Main Layout

The main layout of the MedEx consists of a navigation bar with several tabs to represent the patient data as shown below.


1. Table Browser

Below is the selection panel for the tab: Table browser.
It contains a selector where you can choose which data items will be displayed in the table and what format the table should have i.e. long format or entities as columns. On default, the data is displayed in a long format table. The Visit and the Entities field work similarly as described in the Patient Filter section before but with multiple options to select at a time.


Long format: In this format, the table contains the columns: name_id, measurement (or visit), key and value.


Entities as columns: In this case entities are displayed as columns along with the name_id and measurement (or visit) columns.


The result also has a download button in the top left corner, by pressing this button you can download the table in a csv format.


2. Basic Stats

Database tab: Displays the information about the available data in our database for all types of data items.


Numerical tab: The Basic Statistics numerical table calculates the min/max/average/median/standard deviation/standard error value for all occurrences of each entity or data item.


Categorical and Date tab: The Basic Statistics categorical or date table display the count for the number of patients with the selected visit and entity.


In this case it is also possible to download the table with basic statistics in csv format using the download button.


3. Scatter plot

The below image shows a graphical representation of the data with a scatter plot. This can be applied for only numerical data items and the visit needs to be different if selecting same data items. The result is displayed with linear scale on default


or the user has an option to change the axis to a log scale. The results can also be group based on a categorical entity which can be seen in the image displayed below.


On each graph there is Modebar in the upper right corner . This is a Plotly feature that contains buttons for download chart, zoom, scroll etc. There is also a separate download button which the user can use to download the images in a svg format which provides high resolution and can be used for further analysis.


4. Barchart

The number of occurrences of a categorical data is presented as a percentage or a count.


5. Histogram

It shows distribution of the numerical data items and are grouped by the categorical data. The below image shows the selector panel for the histogram plot with a possibility to change the number of bins.

6. Boxplot
